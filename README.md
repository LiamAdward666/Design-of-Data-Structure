# 介绍
随着全球民航数据的爆发式增长，单一维度的检索已无法满足需求。本项目通过集成多种高级排序算法与图论路径规划算法，旨在展示数据结构在解决复杂工程问题中的核心作用。
1.需求规定
系统必须严格执行任务书要求的四种排序算法与两种图算法，且图结构顶点数必须满足V≥15。
2.运行环境
硬件：兼容x86/x64架构的个人计算机。软件：Windows 10/11或 Linux操作系统。开发工具:CLion/VS Code，编译器支持C++11或更高版本。
3.启动与主界面
运行程序后，系统会自动加载包含15个城市以上的初始航班数据。
屏幕将显示“全球航班查询系统”主菜单，提供8个功能选项（0-7）。
4.功能操作指南
(1)航班查看与排序 (选项1-5)
①查看列表：输入1并回车，系统将按当前顺序显示所有航班信息。
②票价排序：输入2，系统将应用冒泡排序算法，按票价从小到大重新排列列表。
③飞行时长排序：输入3，系统将应用快速排序算法，按飞行时长从小到大排列。
④准点率排序：输入4，系统将应用堆排序算法，按准点率从高到低排列。
⑤航班号排序：输入5，系统将应用基数排序算法，按航班号字母数字顺序排列。
(2)航线咨询与规划 (选项6-7)
①查询所有路径：
i输入6并回车。
ii根据提示输入起点城市和终点城市（例如：Beijing Shanghai）。
iii系统将通过DFS算法输出两地间所有可达的简单路径及其总成本。
②查询最低票价路径：
i输入7并回车。
ii输入起点与终点城市。
iii系统将通过Dijkstra算法计算并输出票价最低的最优方案及转机序列。
5.异常处理与退出
(1)输入错误：若输入菜单以外的数字，系统将提示“无效输入”，请重新选择。
(2)不可达提示：若查询的两个城市之间没有航线，系统将输出“无可行路径”或“城市不存在”。
(3)安全退出：在主菜单输入0即可安全结束程序。
